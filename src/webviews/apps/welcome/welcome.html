启动台

使用启动台将您所有的 GitHub 拉取请求汇总到一个统一的、可操作的列表中，以更好地跟踪进行中的工作、待处理的工作、审查等。保持专注并对最重要的事项采取行动，以保持您的团队畅通无阻。 了解更多

编码建议 编码建议

将您的代码审查从 GitHub 限制性的、仅限评论的反馈风格中解放出来。就像在 Google 文档上建议更改一样，在您已经在编码的地方——您的 IDE 中以及项目中的任何内容上建议代码更改，而不仅仅是 PR 中更改的代码行。 了解更多

云补丁 云补丁

通过从您的工作进度、提交或暂存中创建云补丁并分享生成的链接，轻松且安全地与您的团队成员或其他开发者共享代码更改。使用云补丁来进行早期合作，获取关于方向、方法等的反馈，以减少重做并简化您的工作流程。 了解更多

工作树

通过最小化分支间的上下文切换来高效地进行多任务处理，使您能够同时轻松地在仓库的不同分支上工作。当需要审查一个拉取请求时，避免中断您正在进行的工作。只需创建一个新的工作树并在新的 VS Code 窗口中打开它，所有这些都不会影响您正在进行的工作。

工作空间

利用工作区轻松地将多个仓库分组和管理在一起，可以从任何地方访问，简化您的工作流程。为自己创建工作区或分享（即将在 XU-Git 中推出）给您的团队，以实现更快的入职和更好的协作。 了解更多

可视化文件历史

快速查看文件的演变，包括何时进行了更改、更改的规模有多大以及是谁做的更改。使用它可以快速找到对文件进行了最有影响的更改的时间，或者最适合讨论文件更改及更多信息的人。

集成

简化您的工作流程并通过在包括 GitHub、GitHub Enterprise 、GitLab、GitLab 自托管 、Gitea、Gerrit、Google Source、Bitbucket、Bitbucket Server、Azure DevOps 以及自定义服务器在内的多个 Git 托管平台之间自动链接问题和拉取请求，快速获得洞察。

所有集成都提供自动链接，而与 GitHub 和 GitLab 的深度集成则提供自动链接的详细悬停信息，以及拉取请求、分支和提交之间的相关性，还有用户头像以增加上下文信息。
